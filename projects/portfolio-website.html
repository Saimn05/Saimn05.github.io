Responsive Portfolio Website

Implementation notes

  • Architecture: purely static HTML/CSS/JS served by GitHub Pages, minimizing latency and complexity.
  • Performance strategy:
    • Starfield animation capped and paused on visibilitychange to reduce CPU when tab is hidden.
    • All non-critical images use loading="lazy" + explicit dimensions to prevent layout shift.
    • Single JS file (~small footprint) loaded with defer; no blocking third‑party scripts.
    • CSS theme variables per page (home/projects/experience/contact) swapped via body class, avoiding additional requests.
    • Page transitions implemented with class toggles (no heavy libraries) — average navigation overhead < 2ms.
  • Accessibility:
    • Semantic landmarks (header, main, nav) and descriptive link text.
    • Alt text for all images (mock, data, sketches, prototype, plots).
    • Color palettes chosen for sufficient contrast (target WCAG AA; accent vs background contrast checked).
    • Lightbox is keyboard‑dismissable (Escape) and not auto-focused to avoid focus traps.
  • Data integration: repository dashboard screenshot documents recent deployments (GitHub Pages) and language distribution (HTML/CSS/JS proportions).
  • Resilience: defensive JS guards (null checks, try/catch) ensure partial failures (e.g., GitHub API rate limit) don’t break rendering.
  • Future improvements:
    • Add build step to inline critical CSS and generate a small service worker for offline viewing.
    • Incorporate Lighthouse metrics section (FCP, LCP, CLS) collected from local audits.
    • Offer downloadable design tokens file for color palette reuse.

Structure

Lowercase filenames; assets in images/. Theme variables defined once in style.css and adjusted via body theme classes. Transition classes (page-enter, page-exit) are applied dynamically for a lightweight animated navigation feel.
